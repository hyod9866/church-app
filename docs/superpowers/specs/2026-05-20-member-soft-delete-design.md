# 설계서: 성도 상태 관리 (소프트 삭제) 기능 추가

이 설계서는 성도 정보를 완전히 삭제하는 대신 '이동 또는 교제 안나옴' 상태로 변경하여 목록에서 관리할 수 있는 기능을 정의합니다.

## 1. 목적
- 성도 데이터의 무결성 유지 (출석 기록 등 과거 데이터 보존)
- 실수로 인한 데이터 손실 방지
- '재적' 성도와 '이동/교제 안나옴' 성도의 분리 관리 기반 마련

## 2. 변경 사항

### 2.1 데이터베이스 (SQLite)
- `members` 테이블에 `status` 컬럼 추가
  - 데이터 타입: `TEXT`
  - 기본값: `'active'` (재적)
  - 가능한 값: `'active'`, `'inactive'` (이동/교제 안나옴)

### 2.2 백엔드 API (server.js)
- **DB 초기화 수정:** 테이블 생성 시 `status` 컬럼 포함 또는 기존 테이블 마이그레이션 로직 추가.
- **성도 조회 API 수정:**
  - `GET /api/members/filter`: 기본적으로 `status = 'active'`인 성도만 반환.
  - `GET /api/members/search`: 기본적으로 `status = 'active'`인 성도만 반환.
- **성도 상태 업데이트 API 추가:**
  - `PATCH /api/members/:id/status`: 특정 성도의 상태를 변경.

### 2.3 HTML 프론트엔드 (public/index.html, app.js)
- **UI 변경:** 성도 상세/수정 모달 하단에 '이동 또는 교제 안나옴' 버튼 추가 (빨간색 스타일).
- **로직 추가:** 
  - 버튼 클릭 시 `confirm` 창을 통해 사용자 의사 재확인.
  - API 호출 후 성공 시 모달을 닫고 성도 목록 새로고침.

### 2.4 React 프론트엔드 (App.jsx)
- **상태 관리:** `members` 데이터에 `status` 속성 추가.
- **UI 변경:** 성도 상세 뷰 하단에 '이동 또는 교제 안나옴' 버튼 추가.
- **필터링 로직:** 목록 렌더링 시 `status === 'active'`인 항목만 표시하도록 `filter` 추가.

## 3. 예외 처리 및 보안
- 삭제(상태 변경) 전 사용자 확인 필수.
- 존재하지 않는 성도 ID에 대한 요청 시 404 에러 처리.

## 4. 추후 계획
- '이동/교제 안나옴' 성도들만 따로 모아보고 복구할 수 있는 관리 메뉴 추가.
