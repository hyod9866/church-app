# 심방 현황 관리 기능 설계 문서

## 1. 목적
- 구역별로 심방을 받은 성도와 받지 않은 성도를 한눈에 파악하여 체계적인 심방 관리를 지원한다.
- 기존의 '심방' 모임 기록 데이터를 자동으로 집계하여 관리의 번거로움을 최소화한다.

## 2. 주요 기능
### 2.1 전용 관리 페이지 (`visitation_history.html`)
- **구역별 필터링:** 상단 선택 박스를 통해 특정 구역 성도만 조회 가능.
- **성도별 심방 요약:**
    - 최근 심방 날짜 (예: 2026-05-15)
    - 마지막 심방 후 경과일 (예: 15일 경과)
    - 누적 심방 횟수
- **미심방자 강조:** 심방 기록이 없는 성도를 최상단에 배치하거나 별도 배지로 강조.
- **상세 내역 조회:** 성도 클릭 시 해당 성도의 전체 심방 히스토리(날짜, 비고/간증) 노출.
- **빠른 등록:** 리스트 내 버튼을 클릭하여 즉시 해당 성도에 대한 새로운 심방 기록 작성.

## 3. 기술 설계
### 3.1 프론트엔드
- `visitation_history.html`: 심방 현황 대시보드 레이아웃.
- `js/visitation_history.js`: 
    - 성도 목록 및 심방 데이터를 가져와 구역별로 그룹화 및 정렬.
    - 정렬 로직: 미심방자 -> 심방한 지 오래된 순 -> 이름순.
- 배지 색상:
    - 미심방: 연한 노란색/빨간색 배지 ("기록 없음")
    - 최근 심방: 청록색(Teal) 배지

### 3.2 백엔드 (`server.js`)
- **API 신규 엔드포인트:** `GET /api/visitation/status`
    - 모든 성도 정보와 각 성도별 최신 '심방' 타입 모임 날짜 및 횟수를 JOIN 쿼리로 한 번에 조회.
    - 쿼리 예시: `SELECT m.*, MAX(mt.date) as last_visitation, COUNT(mt.id) as total_count FROM members m LEFT JOIN attendance a ON m.id = a.member_id LEFT JOIN meetings mt ON a.meeting_id = mt.id WHERE mt.type = '심방' GROUP BY m.id`

## 4. 사용자 UI/UX
- **컴팩트 리스트:** 설교 현황 페이지와 유사하게 한 줄에 성도 한 명의 정보가 들어가는 슬림한 디자인.
- **직관적 배지:** "30일 전", "기록 없음" 등 관리자가 행동(심방 계획)을 결정하기 쉬운 텍스트 중심의 지표 제공.

## 5. 성공 기준
- 구역을 선택했을 때, 해당 구역원 중 심방을 안 간 사람이 누구인지 즉시 식별 가능해야 함.
- 성도 이름을 눌러 해당 성도의 과거 심방 기록(메모)을 바로 확인할 수 있어야 함.
