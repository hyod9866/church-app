# 설계서: 성도 목록 필터링 및 정렬 기능 추가

이 설계서는 사용자가 성도 목록을 구역, 성별, 구분(부서)별로 필터링하고, 이름이나 구역순으로 정렬할 수 있는 기능을 정의합니다.

## 1. 목적
- 많은 성도 데이터 중 필요한 인원을 빠르게 찾기 위함
- 모임 성격에 맞는 성도 목록 구성 (예: 자매회 모임 시 자매 필터링)
- 달력 영역을 조정하여 사용자 경험 개선

## 2. 변경 사항

### 2.1 UI/UX 변경 (HTML 및 React 공통)
- **사이드바 (성도 목록 상단):**
  - **필터 드롭다운 3개:** [구역], [성별(B/S)], [구분(부서)]
  - **정렬 버튼:** [이름순/구역순] 토글 또는 드롭다운
- **메인 영역 (달력):**
  - 필터 영역 확보를 위해 달력의 높이 또는 크기를 소폭 조정 (CSS 수정)
- **모임 생성 모달:**
  - 현재 구역 필터 외에 성별 및 구분 필터 추가

### 2.2 데이터베이스 및 API (server.js)
- **`GET /api/members/filter` API 확장:**
  - `gender` (bs), `category` (구분) 파라미터 추가 처리
  - `sort` 파라미터 추가 (name 또는 district)
- **`GET /api/members/search` API 확장:**
  - 검색 시에도 현재 적용된 필터 조건이 유지되도록 쿼리 수정

### 2.3 프론트엔드 로직 (app.js 및 App.jsx)
- **상태 관리:** 현재 선택된 필터와 정렬 기준을 저장하는 상태 변수 추가
- **이벤트 핸들러:** 필터 변경 시 API를 재호출하거나 로컬 데이터를 재필터링/정렬하는 로직 구현

## 3. 상세 설계

### 3.1 필터 옵션
- **구역:** DB의 `district` 컬럼 기준 (예: 전체, 581, 582...)
- **성별:** DB의 `bs` 컬럼 기준 (전체, B, S)
- **구분:** DB의 `category` 컬럼 기준 (전체, 청년회, 은장회, 어머니회...)

### 3.2 정렬 로직
- **이름순:** `name` 컬럼 기준 오름차순
- **구역순:** `district` 컬럼 기준 오름차순 (동일 구역 내에서는 이름순)

## 4. 제약 사항
- 필터 변경 시마다 서버 부하를 줄이기 위해 효율적인 쿼리 작성
- 기존의 '소프트 삭제(status=active)' 조건은 항상 유지

## 5. 추후 계획
- 사용자가 자주 사용하는 필터 조합을 저장하는 '내 필터' 기능 검토
