# 설계 문서: 설교 현황 예정 배지 및 디자인 통일

## 1. 개요
설교 현황 목록에서 아직 진행되지 않은 설교를 직관적으로 파악할 수 있도록 '예정' 배지를 추가하고, 목록의 배지 색상을 메인 달력과 통일하여 시스템 전반의 디자인 일관성을 높입니다.

## 2. 요구사항 및 제약사항
- 오늘 이후의 날짜를 가진 설교에는 '예정' 표시를 추가함.
- 모임 종류별 배지 색상을 `public/js/app.js`에서 사용하는 색상 체계와 동일하게 맞춤.
- 기존의 3000번 포트 기반 HTML/JS 시스템 내에서 작업함.
- 예정 배지는 시인성을 위해 강조된 스타일(파란색 배경)을 사용함.

## 3. 상세 설계

### 3.1. 배지 색상 매핑
`public/js/sermon_history.js` 내에서 다음과 같이 색상 클래스를 할당함:

| 모임 종류 | 배경색 | 글자색 | 테두리 |
| :--- | :--- | :--- | :--- |
| 교구 전체 / 임원 / 기타 | `bg-sky-100` | `text-sky-800` | `border-sky-200` |
| 조모임 | `bg-pink-100` | `text-pink-800` | `border-pink-200` |
| 구역모임 | `bg-orange-100` | `text-orange-800` | `border-orange-200` |
| 형제모임 | `bg-emerald-100` | `text-emerald-800` | `border-emerald-200` |
| 청년모임 | `bg-purple-100` | `text-purple-800` | `border-purple-200` |
| 봉사 | `bg-indigo-100` | `text-indigo-800` | `border-indigo-200` |
| 외부/개인 (설교) | `bg-yellow-100` | `text-yellow-800` | `border-yellow-200` |

### 3.2. '예정' 배지 로직
- **판단 기준**: `new Date(sermon.start)`의 시간이 현재 시점(오늘 자정 기준)보다 큰 경우.
- **스타일**: `bg-blue-600`, `text-white`, `animate-pulse` (반짝이는 효과로 시선 유도).
- **표시 문구**: `예정`

### 3.3. UI 구조 변경
각 설교 카드(`sermon-card`) 내의 배지 영역에 두 개의 배지가 나란히 표시될 수 있도록 수정함:
`[모임종류 배지] [예정 배지(필요시)]`

## 4. 구현 계획
1. `public/js/sermon_history.js`의 `renderSermons` 함수 내 뱃지 생성 로직 수정.
2. 오늘 날짜와 비교하는 유틸리티 로직 추가.
3. 배지 영역의 레이아웃이 깨지지 않도록 `flex` 간격 조정.

## 5. 검증 계획
- 미래 날짜의 데이터를 생성하여 '예정' 배지가 정상적으로 반짝이며 나타나는지 확인.
- 각 모임 종류별로 배지 색상이 달력과 동일하게 바뀌었는지 육안으로 확인.
- 검색 및 필터링 기능이 배지 추가 후에도 정상 작동하는지 테스트.
